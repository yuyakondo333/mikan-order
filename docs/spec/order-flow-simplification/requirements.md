# 注文フロー簡素化 - 要件定義

## 背景・目的

- 現在の注文フローは配送前提で住所7項目+支払い方法を収集しているが、過剰
- 連絡手段はLINE公式アカウント ↔ 顧客個人LINEで十分なため、電話番号は不要
- 受取方法として「取り置き」と「お届け」の2種類を提供する
- お届けの場合は事前振込確認後に発送する（代金引換は廃止）

## 受取方法

### 1. 取り置き（店頭受取）

| 項目 | 内容 |
|------|------|
| 収集情報 | 時間帯のみ（名前はLINEプロフィールから自動取得） |
| 支払い | 店頭で現金払い |
| フロー | 注文受付 → 準備中 → 準備完了 → 受取済 |

#### 時間帯の選択肢

| 値 | 表示ラベル |
|----|----------|
| `morning` | 午前中（9:00〜12:00） |
| `early_afternoon` | 13:00〜15:00 |
| `late_afternoon` | 15:00〜17:00 |

### 2. お届け（配送）

| 項目 | 内容 |
|------|------|
| 収集情報 | 名前、住所（郵便番号・都道府県・市区町村・番地・建物名（任意）） |
| 支払い | 銀行振込（事前入金） |
| フロー | 注文受付 → 入金待ち → 入金確認済 → 準備中 → 発送済 → 配達完了 |

#### 住所フォーム

| フィールド | 必須 | 説明 |
|-----------|------|------|
| 受取人名 | 必須 | 宛名 |
| 郵便番号 | 必須 | 形式: 123-4567 |
| 都道府県 | 必須 | |
| 市区町村 | 必須 | |
| 番地 | 必須 | |
| 建物名・部屋番号 | 任意 | マンション等の場合 |

## 注文ステータス

### 取り置き

```
pending（注文受付）
  → preparing（準備中）
  → ready（準備完了）
  → completed（受取済）
  → cancelled（キャンセル）
```

### お届け

```
pending（注文受付）
  → awaiting_payment（入金待ち）
  → payment_confirmed（入金確認済）
  → preparing（準備中）
  → shipped（発送済）
  → completed（配達完了）
  → cancelled（キャンセル）
```

## 顧客向け画面

### 注文情報入力ページ（現 /address → リニューアル）

1. 受取方法の選択（取り置き / お届け）
2. 受取方法に応じたフォーム表示
   - **取り置き**: 時間帯選択のみ
   - **お届け**: 名前 + 住所入力
3. 注文確定ボタン

### 注文完了後の表示

- **取り置き**: 「ご注文ありがとうございます。準備ができましたらLINEでお知らせします。」
- **お届け**: 「ご注文ありがとうございます。振込先をLINEでご案内しますので、お振込をお願いいたします。」

## 管理画面

### 注文一覧

- 受取方法（取り置き/お届け）を表示
- ステータスに応じたアクションボタン

### ステータス更新ボタン

| 受取方法 | ボタン |
|---------|--------|
| 取り置き | 準備中にする → 準備完了にする → 受取済にする |
| お届け | 入金確認する → 準備中にする → 発送済にする → 配達完了にする |

## 廃止する機能

- 電話番号の収集
- 支払い方法の選択UI（受取方法で自動決定）
- 代金引換

## バリデーション

- 必須項目がすべて入力されるまで「注文を確定する」ボタンを非活性（disabled）にする
- 取り置き: 時間帯が選択されていること
- お届け: 受取人名・郵便番号・都道府県・市区町村・番地がすべて入力されていること

## 非機能要件

- 既存の注文データがある場合はマイグレーションで対応
- LINE連携（LIFF認証）は変更なし
