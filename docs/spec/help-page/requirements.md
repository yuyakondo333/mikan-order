# ヘルプページ 要件定義

## 目的

LIFFアプリ内にヘルプページを設置し、顧客が注文フローや支払い・受取に関する疑問を自己解決できるようにする。

## 背景

### 現状の課題

1. **注文フローの案内がない** — 初めて利用する顧客が操作方法を理解する手段がない
2. **支払い情報がLINE通知に依存** — 銀行振込先はLINEメッセージでのみ案内。見落とすと振込できない
3. **ステータス遷移が不透明** — 注文後に「次に何が起こるか」が顧客に伝わらない
4. **受取方法の違いが分かりにくい** — 取り置きと配送の支払い方法・流れの違いが事前に比較できない

### 期待される効果

- 顧客からの問い合わせ（LINE直接メッセージ等）の削減
- 注文フローの離脱率低下（事前に流れが分かるため安心感）
- 銀行振込の未入金率低下（手順が明確になるため）

## 機能要件

### FR-1: ヘルプページの表示

- `/help` パスでアクセス可能なページを顧客向けルートに追加する
- 既存のcustomerレイアウト（ヘッダー・フッター・LIFF認証）を利用する
- LIFF内ブラウザで正常に表示されること
- 認証不要（未ログインでも閲覧可能）

### FR-2: コンテンツセクション

以下のセクションをアコーディオン（開閉式）で表示する：

#### A. ご注文の流れ
- 商品選択 → カート → 受取方法選択 → 注文確認 → 注文完了 の流れを図解またはステップ表示

#### B. 受取方法について
- **取り置き（店頭受取）**
  - 受取日・時間帯を選択して注文
  - 受取時に店頭で現金払い
  - 準備完了時にLINEでお知らせ
- **お届け（配送）**
  - 配送先住所を入力して注文
  - 銀行振込（事前入金）で支払い
  - 入金確認後に発送、発送時にLINEでお知らせ

#### C. お支払いについて
- 取り置き：受取時に現金払い
- 配送：銀行振込（振込先はLINEでご案内）
- 振込先が届かない場合の対処法

#### D. 注文ステータスについて
- 各ステータスの意味を一覧表示
  - 取り置き：注文受付 → 準備中 → 準備完了 → 完了
  - 配送：注文受付 → 入金待ち → 入金確認済 → 準備中 → 発送済 → 完了

#### E. よくある質問（FAQ）
- 「振込先が届かない」→ LINEのトーク履歴を確認 / 注文履歴で状態確認
- 「受取日を変更したい」→ LINEで連絡
- 「注文したのに確認メッセージが来ない」→ LINEの通知設定確認
- （その他、運用しながら追加）

### FR-3: 導線の設置

- フッターに「ヘルプ」リンクを追加する
- リッチメニューへの追加は本スコープ外（別途対応）

## 非機能要件

### NFR-1: パフォーマンス
- 静的コンテンツのみ（Server Component、DBアクセスなし）
- 即座に表示されること

### NFR-2: アクセシビリティ
- アコーディオンはキーボード操作可能
- 適切な見出し階層（h1 > h2 > h3）

### NFR-3: デザイン
- 既存の顧客向けUIと統一されたデザイン（オレンジ基調）
- モバイルファースト（LIFF内表示が主）

## スコープ外

- リッチメニューの変更（ボタン追加は別作業）
- 管理画面からのヘルプコンテンツ編集機能
- チャットボット的な対話型ヘルプ
- 多言語対応
