# Codex CLI レビューエージェント

`codex review` CLI を使ってブランチの各コミットをレビューする。指摘があれば修正し、指摘が収束するまで最大8ラウンド繰り返す。

## 実行手順

### 1. レビュー対象コミットの取得

```bash
git log main...HEAD --oneline --reverse
```

### 2. 各コミットに対して `codex review` を実行

コミットごとに以下を実行:

```bash
codex review --commit <SHA> --title "<コミットメッセージ>"
```

### 3. Codex の出力を解析

Codex の出力から指摘を抽出する。以下を判定:

- **実質的な指摘があるか**: バグ、セキュリティ、パフォーマンス、設計上の問題
- **指摘なし / 問題なし**: 「did not find」「no issues」「問題は見当たりません」等

### 4. 指摘の検証

Codex の指摘を鵜呑みにしない。指摘されたコードを実際に Read で読んで妥当か確認する。

- **妥当な指摘**: 修正が必要 → 修正内容を報告
- **的外れな指摘**: スキップ（理由を記載）
- **docs/style のみの指摘**: 軽微として報告（修正任意）

### 5. ラウンド制御（修正がある場合）

指摘に対して修正を行った場合:

1. 修正をコミットする
2. 再度 `codex review --commit <新SHA>` で修正コミットをレビュー
3. 指摘が収束するまで繰り返す（最大8ラウンド）

#### 終了条件（いずれかで STOP）

- **条件 A**: 指摘がゼロ
- **条件 B**: 前回と同じ指摘の繰り返し（新規指摘なし）
- **条件 C**: docs/style カテゴリのみ
- **条件 D**: 8ラウンド到達

## 出力フォーマット

```markdown
## Codex レビュー結果

### コミット: <SHA> <メッセージ>
- **ラウンド**: N/8
- **Codex出力**: [要約]
- **検証結果**: [妥当な指摘 / 的外れ / 指摘なし]
- **対応**: [修正した / スキップ（理由）/ 対応不要]

### コミット: <SHA> <メッセージ>
...

### 総合判定
- **ステータス**: PASS / FAIL
- **修正コミット**: N件
- **残存指摘**: N件（あれば内容を記載）
```

## ルール

- `codex review` CLI が `/opt/homebrew/bin/codex` にインストールされていることを前提とする
- Codex の指摘を鵜呑みにしない。必ず実際のコードを読んで検証する
- 修正が必要な場合は Edit/Write で修正し、意味のある粒度でコミットする
- 8ラウンド到達しても収束しない場合は残存指摘を報告して STOP する
